## 一、环境准备

### 1.1 安装 Python

需要 Python 3.10 或更高版本。

**Windows：**
1. 打开 https://www.python.org/downloads/
2. 下载最新版 Python（3.10+），安装时勾选 **"Add Python to PATH"**

**macOS：**
```bash
# 如果没有 Python，用 Homebrew 安装
brew install python
```

**Linux（Ubuntu/Debian）：**
```bash
sudo apt update && sudo apt install python3 python3-pip
```

### 1.2 安装依赖

打开终端（Windows 打开 cmd 或 PowerShell），执行：

```bash
pip3 install numpy matplotlib
```

如果提示 `pip3` 找不到，试试：
```bash
python3 -m pip install numpy matplotlib
# 或 Windows 上：
python -m pip install numpy matplotlib
```

无需编译，无需数据库，纯 Python。

## 四、修改参数运行

### 4.1 参数在哪里

所有参数在 `wheat_t/config.py` 文件中，有详细中文注释。

用任意文本编辑器（记事本、VS Code、PyCharm 等）打开即可修改。

### 4.2 关键参数说明

```python
# ── 光合 OBT 形成（SOLVEG 物理公式）──
K_photo = 10**10.25       # 光合 OBT 形成速率常数 [Bq·s/mol]
                          # ↑ 改大 → 所有器官 OBT 增加

# ── 蛋白周转 ──
K_PROT_TURN = 0.0028      # 叶片蛋白周转速率 [/h]
                          # 半衰期 ≈ 10.3 天
                          # ↑ 改大 → 叶片 OBT 降低（周转快，OBT 被稀释）

# ── 谷粒 OBT 转移 ──
K_REMOB = 0.000146        # canopy→grain OBT 再转移速率 [/h]
                          # ↑ 改大 → 谷粒 OBT 增加

# ── Snapshot receptivity（bell curve）──
RECEPTIVITY_PEAK_FP = 0.012   # bell curve 峰值位置（灌浆进度 0~1）
RECEPTIVITY_SIGMA = 0.008     # bell curve 宽度
RECEPTIVITY_BASELINE = 0.10   # 灌浆刚开始时的 receptivity
```

### 4.3 修改后重新运行

```bash
python3 run_all.py
```

结果会自动更新到 `results/` 目录。

### 4.4 恢复默认值

如果不小心改乱了，从代码包重新解压即可恢复。

## 五、单独运行某个场景

不想跑全部结果，只想看某个特定场景：

### 5.1 方法一：修改 run_all.py

打开 `run_all.py`，找到想运行的部分，注释掉其他部分（在行首加 `#`）。

### 5.2 方法二：写 Python 脚本

创建一个新文件 `my_test.py`：

```python
from wheat_t.config import DIABATE_SCENARIO
from wheat_t.model import SolvegModel

# 复制基线场景，修改参数
params = dict(DIABATE_SCENARIO)
params['airHTO'] = 50000   # 改 HTO 浓度为 50000 Bq/L
params['expH'] = 8         # 改暴露时长为 8 小时

# 运行
m = SolvegModel(params)
result = m.run()
h = result['harvest']

# 输出结果
print(f"谷粒 OBT = {h['grain_OBT_Bq_g']:.1f} Bq/g")
print(f"叶片 OBT = {h['leaf_OBT_Bq_g']:.1f} Bq/g")
print(f"茎   OBT = {h['stem_OBT_Bq_g']:.1f} Bq/g")
print(f"穗   OBT = {h['ear_OBT_Bq_g']:.1f} Bq/g")
print(f"TLI      = {h['TLI_diabate_pct']:.2f}%")
```

然后运行：
```bash
python3 my_test.py
```

### 5.3 运行 DAF 场景

```python
from wheat_t.config import _make_daf_scenario
from wheat_t.model import SolvegModel

# DAF=12 表示开花后第 12 天暴露
params = _make_daf_scenario(daf=12)
m = SolvegModel(params)
result = m.run()
h = result['harvest']
print(f"DAF=12 谷粒 OBT = {h['grain_OBT_Bq_g']:.1f} Bq/g")
```

## 七、模型架构概览

```
大气 HTO (79000 Bq/L)
    ↓ 气孔吸收
叶片 TFWT (水中的氚)
    ↓ 光合固氚 (K_photo × TFWT × A_n)      ← SOLVEG 物理公式
    ↓ 暗代谢固氚 (NU_DARK)                  ← 夜间 OBT 形成
叶片 OBT_str (有机结合氚)
    ↓ 蛋白周转 (K_PROT_TURN)                ← OBT 在叶片内衰减
    ↓ 再转移 (K_REMOB × receptivity)        ← OBT 从叶片到谷粒
    ↓ 衰老转移 (senescence × receptivity)   ← 叶片衰老时 OBT 转到谷粒
谷粒 OBT
```

**核心文件说明：**

| 文件 | 功能 |
|------|------|
| `config.py` | 全部参数定义 + 场景配置 |
| `model.py` | 主循环：气象 → 冠层 → 器官 → 土壤 → 输出 |
| `canopy.py` | 多层冠层：Farquhar 光合 + Ball-Berry 气孔 + 辐射传输 |
| `organs.py` | 器官碳池（叶/茎/穗/谷/根）+ OBT 形成与转移 |
| `plant_water.py` | 植物水分传输（两室水模型） |
| `atmosphere.py` | 气象模块（温度、湿度、风速、辐射） |
| `soil.py` | 土壤 HTO 传输（Van Genuchten 6 层） |

## 八、常见问题

**Q: 运行报错 `ModuleNotFoundError: No module named 'numpy'`**
A: 执行 `pip3 install numpy matplotlib`

**Q: 运行报错 `No module named 'wheat_t'`**
A: 确保在 `paper_code/` 目录下运行，不是在 `wheat_t/` 目录下

**Q: 结果和论文不完全一致**
A: 正常。微小差异（<1%）来自浮点精度。论文四舍五入到整数。

**Q: 想看中间过程，不想等全部跑完**
A: 用快速模式 `python3 run_all.py --quick`，或修改 `run_all.py` 注释掉不需要的部分

**Q: 怎么添加新的场景**
A: 在 `config.py` 中参考 `DIABATE_SCENARIO` 格式定义新场景字典，然后在 `run_all.py` 中调用

**Q: 模型运行很慢**
A: 正常。一个场景约 1-2 分钟（72060 个时间步长）。完整运行约 10 分钟。

## 九、已知局限

1. **暴露期 OBT 绝对值偏差**：模型预测比实测高 150-1800×，原因是单一 OBT_str 池无法同时满足暴露期和 harvest 的动态需求（需双池模型解决）
2. **DAF 绝对值偏差**：趋势正确（peak@DAF=12），但绝对值偏差较大（已确认不需要精确匹配）
3. **适用范围**：适用于气态 HTO 叶片气孔途径急性暴露场景，不适用长期慢性释放或土壤吸收为主的场景
